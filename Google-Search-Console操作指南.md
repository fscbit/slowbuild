# slowbuild.top Google Search Console 操作指南

> 适用版本：2026年6月 | 中英文对照 | 一次性设置 + 日常维护

---

## 一、首次设置（只做一次）

### 1.1 提交 Sitemap

| 英文 | 中文 |
|------|------|
| 左边菜单 → **Sitemaps** | 左边菜单 → **站点地图** |

输入框填写：`sitemap.xml`
点击：**Submit** / **提交**

✅ 看到绿色对勾表示成功。sitemap 告诉 Google 你有哪些页面。

### 1.2 请求收录首页

| 英文 | 中文 |
|------|------|
| 左边菜单 → **URL inspection** | 左边菜单 → **网址检查** |

顶部输入框填写页面网址，按回车：

| 页面 | 网址 |
|------|------|
| 首页 | `www.slowbuild.top` |
| 博客索引 | `www.slowbuild.top/blog/` |
| 八字页 | `www.slowbuild.top/fortune/bazi.html` |

点击：**Request indexing** / **请求编入索引**

✅ 每个重要页面做一次就行，不用反复提交。

## 二、什么时候需要操作

### 🔴 需要立刻处理

**场景：Console 首页出现红色报错**

| 英文 | 中文 |
|------|------|
| 左边 → **Shopping** → **Product snippets** | 左边 → **购物** → **产品摘要** |
| 或 左边 → **Enhancements** | 或 左边 → **增强功能** |

点进去看具体报错 → 告诉我 → 我修代码 → 修完后回来：

点击右上角 **Validate fix** / **验证修复**

---

### 🟡 改完网站后顺便做

**场景：新增了页面、修改了产品数据**

| 英文 | 中文 |
|------|------|
| 左边 → **URL inspection** | 左边 → **网址检查** |
| 输入新页面网址 → 回车 | 输入新页面网址 → 回车 |
| 点 **Request indexing** | 点 **请求编入索引** |

✅ 只提交新页面，已收录的老页面不用管。

---

### 🟢 每周看一眼（不用操作）

| 看什么 | 英文路径 | 中文路径 |
|--------|----------|----------|
| 索引了多少页 | 左边 → **Indexing** → **Pages** | 左边 → **索引** → **页面** |
| 搜索点击量 | 左边 → **Performance** | 左边 → **效果** |
| 谁搜到了你 | Performance → 往下翻看 **Queries** | 效果 → 往下翻看 **查询** |

📊 绿色数字在涨 = 正常，不用操作。数字大跌或归零 = 有问题，找我。

## 三、完整路径速查表

| 操作 | 英文点击路线 | 中文点击路线 | 何时做 |
|------|-------------|-------------|--------|
| 提交 sitemap | 左边 **Sitemaps** → 输入框 → Submit | 左边 **站点地图** → 输入框 → 提交 | 首次 / 加了新页面 |
| 收录首页 | 左边 **URL inspection** → 输入网址 → Request indexing | 左边 **网址检查** → 输入网址 → 请求编入索引 | 首次 / 改了首页 |
| 验证修复 | **Shopping** → **Product snippets** → Validate fix | **购物** → **产品摘要** → 验证修复 | Console 报红时 |
| 看索引数量 | 左边 **Indexing** → **Pages** | 左边 **索引** → **页面** | 每周看一次 |
| 看搜索数据 | 左边 **Performance** | 左边 **效果** | 每周看一次 |

## 四、常见情况对照

| Console 显示 | 意思 | 怎么办 |
|-------------|------|--------|
| "Processing data, please check again in a day" | 数据还在生成 | 等 1-2 天再看 |
| "URL is not on Google" | 这个页面没被搜录 | 点 Request indexing |
| 红色报错有数字 | 结构化数据有问题 | 截图给我，我修代码 |
| 绿色数字在涨 | 一切正常 | 什么都不做 |

---

## 五、⚠️ 不需要做的事情

- ❌ 不要每天提交 sitemap
- ❌ 不要每天点 Request indexing
- ❌ 不要反复点 Validate fix（要等它验证完，可能几天）
- ❌ 不要用百度站长工具（网站面向海外）
- ❌ 不要因为数字没涨就焦虑（新站 3-6 个月才有稳定排名）
